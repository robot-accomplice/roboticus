# v1.0.1 Release Notes

**Date**: 2026-04-12
**PR**: #25 (fix), #26 (release merge)
**Tag**: `v1.0.1`

---

## Summary

Memory recall was fundamentally broken — asking Duncan about past topics
produced confabulated responses instead of actual stored memories. Root cause
was a chain of 5 compounding issues spanning the retrieval pipeline, prompt
injection, memory indexing, tool surface, and LLM serialization.

Additionally, multi-turn tool use with remote providers (OpenAI, Moonshot/Kimi)
was silently broken due to a message serialization bug.

## Changes

### Bug Fixes (5)

| Fix | Files | Impact |
|-----|-------|--------|
| FTS5 MATCH not used in episodic retrieval | `retrieval.go` | Old memories invisible — query joined FTS table but never used MATCH clause |
| Full memory dump injected into context | `daemon.go`, `retrieval.go` | Model treated injected block as complete memory — never called recall tools |
| Memory index polluted with tool output noise | `memory_recall.go` | Top-20 index dominated by `bash: ...` and `get_runtime_context: ...` entries |
| Confidence inflation (all entries at 1.0) | `memory_recall.go`, `schema.go` | 5,545 entries at max confidence made index ordering meaningless |
| OpenAI tool_call_id serialization | `client_formats.go` | Go's `omitempty` dropped `content` field from assistant tool-call messages, causing provider rejection on second ReAct turn |

### New Features (3, beyond Rust parity)

| Feature | Files | Description |
|---------|-------|-------------|
| `search_memories(query)` tool | `memory_recall.go`, `daemon.go` | FTS5 + LIKE fallback search across all 5 memory tiers — Rust never implemented this |
| Query-aware memory index | `memory_recall.go`, `daemon.go` | `BuildMemoryIndex()` surfaces topic-matched entries in first 1/3 of injected index |
| Confidence normalization | `memory_recall.go`, `schema.go` | Default 0.8 (was 1.0), incremental +0.1 on recall (was: binary reset to 1.0) |

### Behavioral Improvements (2)

| Change | Files | Description |
|--------|-------|-------------|
| Anti-confabulation behavioral contract | `prompt.go` | Explicit rule: never fabricate memories; use tools or be honest |
| Two-stage injection (Rust parity) | `daemon.go`, `retrieval.go` | Only working memory + ambient recency injected; everything else via index |

### Documentation (1)

| Doc | Description |
|-----|-------------|
| `docs/memory-architecture-spec.md` | Comprehensive spec derived from exhaustive 4-agent analysis of Rust v0.11.4 |
