## Summary

Comprehensive hardening release: security-first fixes (Store.DB() removed, wallet passphrase keystore-only, delivery queue recovery), pipeline refactored from 874-line monolith to 16 named stage methods, two new apps (Eastern/Western Philosophy), two new plugins (Codex CLI, Web Research), dashboard settings overhaul (16 hand-crafted sections, struct-driven schema), theme texture registry (site-hosted images with local caching), soak-appropriate test timeouts, and cache guards for unparsed tool calls. All 32 planned items shipped — nothing deferred.

## Phase 1 — Security (P0)

| Item | Description |
|------|-------------|
| Store.DB() removed | Raw `*sql.DB` accessor deleted — all callers migrated to `Store.ExecContext`/`QueryContext`/`QueryRowContext`. Architecture test prevents re-addition. |
| Wallet keystore-only | `ROBOTICUS_WALLET_PASSPHRASE` env var removed. Passphrase resolution: keystore lookup → migration on first use → error if empty. `walletMachinePassphrase()` deleted. |
| Delivery queue recovery | `recoverStrandedDeliveries()` runs at startup — resets `in_flight` → `pending` with 30s retry delay. |
| OAuth context leak | `server.Shutdown(context.Background())` → `server.Shutdown(ctx)` with goroutine monitoring ctx cancellation. |

## Breaking Changes

- `Store.DB()` removed — callers must use `Store.ExecContext`/`QueryContext`/`QueryRowContext`
- `ROBOTICUS_WALLET_PASSPHRASE` env var no longer supported — use keystore
- `security.workspace_only` (top-level) deprecated — use `security.filesystem.workspace_only`
- `APIKeyEnv`, `TokenEnv`, `PasswordEnv` config fields removed — use keystore
