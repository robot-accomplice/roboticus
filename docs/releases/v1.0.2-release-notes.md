# v1.0.2 Release Notes

**Date**: 2026-04-12
**PRs**: #28, #29, #30, #31
**Tag**: `v1.0.2`

---

## Summary

Comprehensive parity sweep: bot command parity with Rust, FTS coverage for all
memory tiers, auto-indexing at ingestion, security claim wiring, CLI command
parity, model baselining for memory recall, and banner version fix.

## Changes

### Bot Command Parity (PR #30)

| Command | Description | Authority |
|---------|-------------|-----------|
| `/model` | Show/set/reset model override | Creator for set/reset |
| `/models` | List primary + fallbacks + provider status | None |
| `/breaker` | Circuit breaker status + reset | Creator for reset |
| `/retry` | Replay last assistant response | None |

Also: `@bot_name` stripping, authority gating, unified BotCommandHandler
replacing the old `tryBotCommand()` switch. Dispatch moved to after session
resolution (Stage 4b).

### Memory System (PR #31)

- FTS5 triggers for procedural + relationship tiers (was: LIKE fallback only)
- Table name normalization: `memory_fts` uses full names (`episodic_memory` not `episodic`)
- Migration 037: normalizes existing data + recreates triggers
- Auto-indexing in ingest path: `autoIndex()` after every `store_*` call

### Security Architecture Gaps Closed (PR #31)

- **Gap 1**: SecurityClaim resolvers confirmed wired (session stores claim + trace)
- **Gap 2**: API routes now construct `ChannelClaimContext` (was: `Claim: nil`)
- **Gap 3**: HMAC trust boundary instructions in system prompt

### Model Baselining (PR #28)

- `IntentMemoryRecall` intent class (5th, beyond Rust parity)
- 5 exercise prompts testing memory tool selection + anti-confabulation
- Confabulation penalty: 0.4x on final score for fabricated memories
- Baselines for 9 models (gemma4: 0.25, enabling automatic router escalation)

### CLI Parity (PR #31)

- `channels health [platform]` — adapter connectivity check
- `channels connect/disconnect` — documented stubs
- `channels` aliased as `integrations`
- `update providers` / `update skills` — standalone commands
- `plugins` aliased as `apps`

### Infrastructure (PR #29)

- Banner version now injected via ldflags (was: always showing "vdev")
