## Summary

Agentic retrieval architecture: systemic redesign of the memory retrieval pipeline around a 13-layer reference architecture for agentic AI systems. Replaces the previous "embed query → search vector DB" approach with intent-driven routing, evidence filtering, query decomposition, structured context assembly, and post-turn reflection. Working memory now persists across restarts.

Guiding principle: **Retrieve broadly, reason narrowly, act cautiously, learn continuously.**

## Phase 1 — Retrieval Substrate

| Item | Description |
|------|-------------|
| HNSW misnomer fix | `hnsw.go` → `vector_brute.go`. `HNSWIndex` → `BruteForceIndex`. All callers updated. The name now matches the O(n) implementation. |
| VectorIndex interface | `vector_index.go` — abstraction over search impls. BruteForce, HNSWGraph, PartitionedIndex all satisfy it. |
| float32 throughout | Embedding storage + computation switched from float64 → float32. Memory per 768-dim entry: ~6KB → ~3KB. |
| BM25 scoring | SQLite FTS5 native `bm25()` replaces positional rank decay. Cross-leg deduplication prevents duplicate scoring. |
| Adaptive hybrid weight | FTS-vs-vector blend adapts to corpus size: 0.7 at <1K entries → 0.3 at >10K entries. |
| Collapse metrics | ScoreSpread, AvgFTSScore, AvgVectorScore, CorpusSize, HybridWeight on RetrievalMetrics. |
| Pure Go HNSW | 350-LOC HNSW graph with diversity heuristic. 100% recall at 1K. Known limitation at 10K/768-dim. |
| Tier-partitioned index | Hot (working+episodic) / warm (semantic+procedural+relationship) partitions. |
| Config cleanup | Removed `AnnIndex` toggle. Renamed `HybridWeight` → `HybridWeightOverride`, `ANNActivationThreshold` → `VectorIndexThreshold`. |

## Phase 2 — Agentic Architecture Layers

| Item | Layer | Description |
|------|-------|-------------|
| Retrieval Router | 8 | Intent-driven tier selection. 11 routing plans based on query intent + keyword signals. Working memory never routed — always direct-injected. |
| Reranker | 11 | Evidence filter that discards, not reorders. Authority boost (1.5x canonical), recency penalty (0.8x >30 days), collapse detection (spread < 0.05 → cap to 3). |
| Query Decomposer | 2 | Splits compound queries at conjunctions/semicolons/question marks. Each subgoal classified to target tier. |
| Procedural Upgrade | 5 | Schema enriched: preconditions, error_modes, last_used_at, avg_duration_ms, context_tags. Learning extractor wired into post-turn for procedure detection. |
| Reflection | 16 | Structured episode summaries (goal/actions/outcome/learnings) generated post-turn. Stored as episodic_memory with importance=8 for consolidation promotion. |
| Working Memory Persistence | — | Shutdown: persist working memory. Startup: vet entries (discard stale/low-importance, retain goals/decisions). Like waking up after sleep. |
| Structured Context Assembly | 12 | Output format: [Working State] + [Retrieved Evidence] + [Gaps] + [Contradictions]. Replaces undifferentiated text blobs. |

## Known Limitations

1. **HNSW at 10K/768-dim**: Recall degrades to 31-70% depending on parameters. Brute-force is competitive at partition sizes the architecture targets (~10K). Revisit after heap optimization and real user scale data.
2. **manager.go at 797 lines**: Pre-existing — exceeds 500-line cap. Split deferred to follow-up.
3. **Parallel retrieval**: Tiers still queried sequentially by the router. True parallel retrieval is v1.1.0.
4. **LLM-based reranking**: v1.0.5 uses score-based filtering. Cross-encoder reranking is v1.1.0.
5. **Graph persistence**: knowledge.go remains ephemeral. Graph-augmented retrieval is v1.1.0.
