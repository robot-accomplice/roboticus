# v1.0.8 Roadmap

## RCA-Derived Efficacy Scoring

- **Correction burden as a first-class efficacy metric**
  - derive a per-model and per-turn correction-burden artifact from canonical
    RCA instead of treating all successful outcomes as equal
  - include at least:
    - normalization corrections
    - blocked malformed tool calls
    - guard retries
    - verifier retries
    - replay suppression
    - degraded final status
  - keep framework-owned corrections separate from model-owned error burden so
    we do not punish a model for our own defects
  - make the artifact reusable for:
    - benchmark scoring
    - routing recommendations
    - operator guidance
    - future self-healing / recommendation logic

## Windows Confidence Expansion

- **Add Windows to the testing matrix in phased confidence order**
  - phase 1:
    - add `windows-latest` to the CI compile/unit matrix
    - keep the goal narrow: catch compile breaks, path-separator drift,
      executable-suffix mistakes, and obvious platform assumptions
  - phase 2:
    - add a Windows smoke/install gate so release confidence includes basic
      runtime and upgrade truth on Windows, not just compilation
  - phase 3:
    - add meaningful Windows behavioral/soak coverage only after the shell,
      process-control, and packaging seams have been cleaned up enough to make
      those signals trustworthy
  - required audit surfaces:
    - shell/script Unix assumptions (`bash`, `sed`, `mktemp`, `sha256sum`,
      signals, process groups)
    - path canonicalization and separator behavior
    - daemon control / install / upgrade paths
    - packaging and installer verification

## CI Test-Surface Hygiene

- **Split and profile the slow CI test surface, especially on ubuntu under `-race`**
  - current observed evidence from the `v1.0.7` release path:
    - macOS test job: `8m 12s`
    - ubuntu test job: `20m 25s`
  - identify the worst package-level outliers in CI timing, with immediate
    attention to:
    - `internal/api/routes`
    - `internal/agent/memory`
    - `internal/db`
    - `internal/pipeline`
  - produce a top-slowest-tests inventory from the ubuntu runner instead of
    treating the wall-clock gap versus macOS as generic runner noise
  - move integration-like and soak-like cases out of broad package test runs
    when they do not belong in ordinary unit/package gates
  - preserve deterministic unit coverage in-package while narrowing the heavy
    environment-sensitive cases into explicit targeted jobs
  - success criteria:
    - ubuntu/macOS timing differences are explainable by scoped workloads, not
      accidental package bloat
    - ordinary PR test jobs stop carrying hidden integration/soak weight
    - CI duration is reduced without weakening release-confidence coverage
