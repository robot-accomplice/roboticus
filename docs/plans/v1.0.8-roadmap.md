# v1.0.8 Roadmap

## RCA-Derived Efficacy Scoring

- **Correction burden as a first-class efficacy metric**
  - derive a per-model and per-turn correction-burden artifact from canonical
    RCA instead of treating all successful outcomes as equal
  - include at least:
    - normalization corrections
    - blocked malformed tool calls
    - guard retries
    - verifier retries
    - replay suppression
    - degraded final status
  - keep framework-owned corrections separate from model-owned error burden so
    we do not punish a model for our own defects
  - make the artifact reusable for:
    - benchmark scoring
    - routing recommendations
    - operator guidance
    - future self-healing / recommendation logic
